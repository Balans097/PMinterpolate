## Обзор

**parallel_minterpolate** — утилита командной строки для ускорения процесса интерполяции кадров длинных видеофайлов. Фильтр `minterpolate` в FFmpeg является однопоточным, поэтому на длинных видео обработка занимает значительное время. Программа автоматизирует стандартный обходной путь: делит исходное видео на части, обрабатывает их параллельно и склеивает обратно в один файл.

Программа генерирует и немедленно запускает батч-файл Windows (`.bat`), который выполняет все необходимые FFmpeg-команды.

---

## Как это работает

Весь процесс состоит из трёх последовательных этапов:

```
Исходное видео
      │
      ▼
┌─────────────┐
│  Разбивка   │  ffmpeg -f segment   →   output000.mp4 … outputN.mp4
└─────────────┘
      │
      ▼
┌─────────────────────────────────────────────┐
│  Параллельная интерполяция (N задач сразу)  │
│  TASK 1: output000.mp4 → output000.60fps.mp4│
│  TASK 2: output001.mp4 → output001.60fps.mp4│
│  …                                          │
└─────────────────────────────────────────────┘
      │
      ▼
┌─────────────┐
│  Склейка    │  ffmpeg -f concat    →   final.mp4
└─────────────┘
```

**Этап 1 — Разбивка.** Исходный файл нарезается на `N` равных по времени фрагментов без перекодирования (`-c copy`), что выполняется очень быстро.

**Этап 2 — Интерполяция.** Каждый фрагмент обрабатывается отдельным процессом FFmpeg с фильтром `minterpolate`. Все процессы запускаются одновременно, загружая доступные CPU-ядра.

**Этап 3 — Склейка.** Интерполированные фрагменты объединяются в итоговый файл `final.mp4` без повторного кодирования.

## Требования

| Компонент | Версия | Примечание |
|---|---|---|
| **Windows** | 7 / 10 / 11 | Только Windows (батч-файл `.bat`) |
| **FFmpeg** | любая актуальная | Должен быть доступен в `PATH`. Включает `ffprobe`. |
| **Nim** | 1.6+ | Только для сборки из исходников |
| **Входной файл** | MP4 | Рекомендуется MP4; другие контейнеры могут работать |

### Проверка наличия FFmpeg

```cmd
ffmpeg -version
ffprobe -version
```

Если команды не найдены — [скачайте FFmpeg](https://ffmpeg.org/download.html) и добавьте папку `bin` в переменную окружения `PATH`.

## Установка и сборка

### 1. Получить исходный код

```cmd
git clone https://github.com/your-username/parallel-minterpolate-nim.git
cd parallel-minterpolate-nim
```

Или просто скачайте файл `parallel_minterpolate.nim`.

### 2. Установить компилятор Nim

Скачайте установщик с [nim-lang.org](https://nim-lang.org/install_windows.html) и следуйте инструкции. После установки убедитесь, что `nim` доступен в `PATH`:

```cmd
nim --version
```

### 3. Скомпилировать

```cmd
nim c -d:release parallel_minterpolate.nim
```

После компиляции появится исполняемый файл `parallel_minterpolate.exe`.

#### Флаги компиляции (опционально)

| Флаг | Описание |
|---|---|
| `-d:release` | Оптимизация скорости, отключение отладочных проверок |
| `--opt:size` | Уменьшение размера бинарника |
| `-d:strip` | Удаление отладочных символов |

```cmd
nim c -d:release --opt:speed -d:strip parallel_minterpolate.nim
```

## Использование

```
parallel_minterpolate.exe <inputVideo> --split <N> [OPTIONS]
```

Программа выводит информацию о видео, создаёт директорию вывода, генерирует вспомогательные файлы и немедленно запускает обработку.

---

## Аргументы и опции

### Позиционный аргумент

| Аргумент | Описание |
|---|---|
| `inputVideo` | Путь к входному видеофайлу (MP4). **Обязателен.** |

### Опции

| Опция | Тип | По умолчанию | Описание |
|---|---|---|---|
| `--split N` | `int` | — | Количество частей для параллельной обработки. **Обязателен.** Рекомендуется равным числу логических ядер CPU. |
| `-o`, `--outputDir NAME` | `string` | `output` | Имя директории для промежуточных и итогового файлов. Создаётся автоматически. |
| `--fps N` | `int` | `60` | Целевая частота кадров после интерполяции. |
| `--shutdown` | флаг | выкл. | Выключить компьютер по завершении всех задач. Удобно для ночной обработки. |
| `-h`, `--help` | флаг | — | Показать справку и выйти. |

## Примеры

### Базовый запуск

```cmd
parallel_minterpolate.exe input.mp4 --split 4
```

Разбить `input.mp4` на 4 части, интерполировать каждую до 60 fps, склеить в `output\final.mp4`.

### Указать целевой FPS

```cmd
parallel_minterpolate.exe input.mp4 --split 4 --fps 120
```

Интерполяция до 120 fps.

### Указать директорию вывода

```cmd
parallel_minterpolate.exe input.mp4 --split 8 --outputDir my_project
```

Все файлы будут созданы в директории `my_project\`.

### Ночная обработка с выключением

```cmd
parallel_minterpolate.exe big_video.mp4 --split 6 --fps 60 --shutdown
```

По завершении компьютер выключится автоматически.

### Полный набор опций

```cmd
parallel_minterpolate.exe C:\Videos\clip.mp4 --split 8 --fps 120 --outputDir C:\Videos\clip_out --shutdown
```

## Структура выходных файлов

После запуска с `--split 3` и `--outputDir output` директория будет выглядеть так:

```
output\
├── list.txt            # Список фрагментов для ffmpeg concat
├── run.bat             # Сгенерированный батч-файл (запускается автоматически)
│
├── output000.mp4       # Фрагмент 1 (исходный, без перекодирования)
├── output001.mp4       # Фрагмент 2
├── output002.mp4       # Фрагмент 3
│
├── output000.60fps.mp4 # Фрагмент 1 после интерполяции
├── output001.60fps.mp4 # Фрагмент 2 после интерполяции
├── output002.60fps.mp4 # Фрагмент 3 после интерполяции
│
└── final.mp4           # Итоговый склеенный файл ← результат
```

> **Совет.** После получения `final.mp4` промежуточные файлы (`output***.mp4`) можно удалить — они больше не нужны.

## Архитектура кода

### Типы данных

#### `Config`

```nim
type
  Config = object
    inputVideo : string   # Путь к входному файлу
    split      : int      # Количество частей
    outputDir  : string   # Директория вывода
    fps        : int      # Целевой FPS
    shutdown   : bool     # Выключить ПК после завершения
```

Структура хранит все параметры, переданные пользователем через командную строку. Создаётся единожды процедурой `parseArgs()` и передаётся в `main()` по значению.

---

### Процедуры

#### `zeroPad(n, width: int): string`

Дополняет целое число `n` ведущими нулями до длины `width`.

```nim
zeroPad(3, 3)   # → "003"
zeroPad(12, 3)  # → "012"
zeroPad(100, 3) # → "100"
```

Используется для формирования имён файлов вида `output003.mp4`.

---

#### `secondsToHHMMSS(totalSecs: int): string`

Конвертирует целое количество секунд в строку формата `HH:MM:SS`.

```nim
secondsToHHMMSS(3661)  # → "01:01:01"
secondsToHHMMSS(90)    # → "00:01:30"
```

Результат передаётся в параметр `-segment_time` команды FFmpeg.

---

#### `getVideoDurationSeconds(videoPath: string): int`

Вызывает `ffprobe` для получения длительности видео в секундах. Завершает программу с кодом `1` и сообщением об ошибке в следующих случаях:

- `ffprobe` не найден или вернул ненулевой код выхода;
- вывод `ffprobe` не удаётся преобразовать в число.

Внутренняя команда:

```
ffprobe -v error -show_entries format=duration
        -of default=noprint_wrappers=1:nokey=1 "<videoPath>"
```

---

#### `parseArgs(): Config`

Разбирает аргументы командной строки с помощью стандартного модуля `std/parseopt`. Проверяет:

- наличие обязательного позиционного аргумента `inputVideo`;
- наличие обязательной опции `--split` со значением `> 0`;
- существование входного файла на диске.

При любой ошибке выводит сообщение в `stderr` и завершает программу с кодом `1`.

---

#### `main()`

Точка входа в программу. Выполняет следующие шаги по порядку:

1. Вызывает `parseArgs()` и получает конфигурацию.
2. Определяет длительность видео через `getVideoDurationSeconds()`.
3. Вычисляет продолжительность каждой части: `partsSeconds = videoSeconds div split`.
4. Создаёт директорию вывода (`createDir`), если она не существует.
5. Записывает `list.txt` — список интерполированных фрагментов для `ffmpeg concat`.
6. Записывает `run.bat` — батч-файл с тремя этапами обработки.
7. Меняет рабочую директорию на директорию вывода и запускает `run.bat` через `startProcess`.

## Описание генерируемого батч-файла

`run.bat` генерируется автоматически и состоит из трёх секций.

### Секция 1 — Разбивка видео

```bat
ffmpeg -i "C:\path\to\input.mp4" -c copy -map 0 -segment_time 00:05:00 -f segment -reset_timestamps 1 output%03d.mp4
```

| Параметр | Значение |
|---|---|
| `-c copy` | Копирование потоков без перекодирования (быстро) |
| `-map 0` | Включить все потоки (видео, аудио, субтитры) |
| `-segment_time` | Длительность каждого фрагмента в формате `HH:MM:SS` |
| `-f segment` | Формат вывода — сегментирование |
| `-reset_timestamps 1` | Сброс временных меток для корректной склейки |

### Секция 2 — Параллельная интерполяция

```bat
(
  start "TASK 1" ffmpeg -i output000.mp4 -crf 10 -vf "minterpolate=fps=60:mi_mode=mci:mc_mode=aobmc:me_mode=bidir:vsbmc=1" output000.60fps.mp4
  start "TASK 2" ffmpeg -i output001.mp4 -crf 10 -vf "minterpolate=fps=60:mi_mode=mci:mc_mode=aobmc:me_mode=bidir:vsbmc=1" output001.60fps.mp4
  ...
) | pause
```

Параметры фильтра `minterpolate`:

| Параметр | Значение | Описание |
|---|---|---|
| `fps` | `N` (из `--fps`) | Целевая частота кадров |
| `mi_mode` | `mci` | Режим компенсации движения (Motion Compensated Interpolation) |
| `mc_mode` | `aobmc` | Адаптивная взвешенная компенсация перекрытия блоков |
| `me_mode` | `bidir` | Двунаправленный поиск движения |
| `vsbmc` | `1` | Компенсация движения переменного размера блоков |

Конструкция `(...) | pause` заставляет командную строку ждать завершения всех дочерних процессов перед переходом к следующему шагу.

### Секция 3 — Склейка

```bat
timeout /t 3 /nobreak > nul
ffmpeg -f concat -safe 0 -i list.txt -c copy final.mp4
```

Склейка выполняется без перекодирования (`-c copy`), поэтому качество не теряется.

### Секция 4 (опционально) — Выключение

```bat
timeout /t 3 /nobreak > nul
shutdown /s /f /t 0
```

Добавляется только при передаче флага `--shutdown`.

## Ограничения

- **Только Windows.** Батч-файл использует команды `start`, `pause`, `timeout`, `shutdown`, специфичные для Windows CMD.
- **Только MP4.** Входные файлы должны быть в контейнере MP4. Другие форматы могут работать некорректно при сегментировании.
- **Нет прогресс-бара.** Ход выполнения отображается непосредственно в окнах FFmpeg.
- **Нагрузка на CPU.** `N` параллельных задач создают `N`-кратную нагрузку. Рекомендуется устанавливать `--split` равным количеству физических или логических ядер CPU.
- **Дисковое пространство.** В процессе работы временно требуется ~3× размера исходного файла (фрагменты + интерполированные фрагменты + итог).

## Отличия от оригинальной Python-версии

| Аспект | Python-версия | Nim-версия |
|---|---|---|

| **Установка** | `pip install opencv-python` | Только компилятор Nim |

## Устранение неполадок

### `ffprobe failed` при запуске

**Причина:** FFmpeg не установлен или не добавлен в `PATH`.

**Решение:** Установите FFmpeg и убедитесь, что папка `bin` (содержащая `ffmpeg.exe` и `ffprobe.exe`) добавлена в системную переменную `PATH`. Проверьте: `ffprobe -version` в командной строке.

---

### Батч-файл открывается и сразу закрывается

**Причина:** Одна из FFmpeg-команд завершилась с ошибкой.

**Решение:** Запустите `run.bat` вручную из командной строки (`cmd.exe`), чтобы увидеть сообщение об ошибке.

---

### `final.mp4` содержит рывки на стыках фрагментов

**Причина:** Некорректные временные метки при сегментировании.

**Решение:** Убедитесь, что в батч-файле присутствует флаг `-reset_timestamps 1`. Также попробуйте уменьшить значение `--split` для более длинных фрагментов.

---

### Слишком высокая нагрузка на CPU / зависание системы

**Причина:** `--split` установлен слишком высоким.

**Решение:** Установите `--split` равным числу логических ядер минус 1–2, чтобы оставить ресурсы для системы. Узнать количество ядер: `echo %NUMBER_OF_PROCESSORS%` в CMD.

---

### Файл `output\final.mp4` не создан

**Причина:** Одна или несколько задач интерполяции не завершились до запуска склейки.

**Решение:** Убедитесь, что все окна `TASK N` закрылись перед тем, как началась склейка. Конструкция `(...) | pause` должна обеспечивать ожидание, но на некоторых системах поведение может отличаться. В этом случае запустите финальную команду вручную:

```cmd
cd output
ffmpeg -f concat -safe 0 -i list.txt -c copy final.mp4
```
